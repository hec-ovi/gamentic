# Changelog

Notable changes to gamentic, newest first. No version numbers yet: this moves fast, so entries are dated and the README always describes the current state.

## 2026-07-20

### The turn goes live: real streaming and a stop button
- The turn is no longer a black box you wait out. While `POST /action` resolves, the per-game SSE stream (the same one that announced media) now mirrors the work as it happens: a phase line says who is working (reading your words / the narrator is writing / Mara is replying), every beat appears the instant the engine stores it, and narrator prose plus character say/whisper text GROW on screen at real generation speed, token by token. The typewriter simulation retires for streamed text (the model's own pace is the pacing); it stays as the fallback for SSE-less clients and late media beats, which behave exactly as before.
- A Stop button rides the working line. `POST /games/{gid}/stop` cancels the whole turn: the in-flight LLM call aborts (llama.cpp frees the slot on disconnect) and the turn's transaction rolls back, so a stopped turn never happened - no beats, no player echo, no clock tick, and the typed words return to the composer. Measured live: stop answered in 10ms and the running turn ended within 20ms of it.
- Under the hood, `llm.chat()` gained a streaming transport (opted into per call via `on_delta`/`cancel`, `LLM_STREAM=false` is the kill-switch back to blocking), and a new `streamscrub.py` makes streaming safe against every model leak the batch scrubbers already catch: the accumulated raw text is re-scrubbed per fragment and only the stable prefix is released, with hold-back for open think-spans, tool debris, fences, markup and screenplay/scaffold line shapes. The stored story is byte-identical with streaming on or off: beats still land through the same emit -> add_beat -> one-commit path, the batch scrub stays the single source of truth, and the live feed is a display-plane mirror the POST response reconciles (a failed turn takes its streamed content back).
- Verified against the real model (26B heretic on llama.cpp Vulkan): four live turns including a mid-generation stop and a whisper exchange whose 146 streamed fragments all routed private with zero public leakage. New tests: the streaming transport (7), scrubber equivalence at every chunk split point (104 parameterized), live events + stop semantics through the real routes (9), and 8 frontend component tests (streaming, stop-cancel, echo dedup, private routing, badge exactness); both suites fully green (644 backend, 226 frontend).
- Play-feel fixes from the first live session: the composer opens in SAY mode (order say / do / look, owner request); the player's echoed line no longer shows twice while the turn generates (the live canonical echo replaces the optimistic one on arrival, public story and whisper thread alike); the unread whisper badge counts literal messages only, never actions, gift receipts or memory events; locked chips (scene actions, ways out, character offers) now LOOK locked mid-turn (they were already unclickable, but carried no disabled style and still glowed on hover); the settings icon button's chamfer cut shrank to 4px so it reads as a chamfer again instead of a deformed box.
- `./up.sh harness [llm-url] [model]`: the whole stack except llm-text. ComfyUI, image-api, Maya voice and voice-api all run locally; only text inference goes to an external OpenAI-compatible server (default `http://host.docker.internal:8090/v1`, the llama-vulkan-strix stack). Proven live: a dockerized turn against the host's llama server streamed 55 fragments end to end. Also: the llm-text service now runs with `--cache-ram 0`, working around a crash in current rolling llama.cpp builds (the server dies on its SECOND request in `prompt_save` on slot reuse, verified on b10068), and the orchestrator container can reach the host via `host.docker.internal` on Linux.

### The setup faces replace the admin panel
- Two pre-flight setup faces, one shared schema (`infra/setup/schema.js`): `./gamentic-setup` (stdlib-python CLI: interactive wizard, agent-friendly flags, a `doctor` subcommand that checks docker, GPU nodes, model files, ports and config pitfalls) and `setup.html` (a static page you double-click; zero network calls, keys never leave the machine; saves your `.env` directly on Chromium, downloads it elsewhere). Both ask the same questions, write a complete `.env`, and preserve any keys they do not know. `.env.example` is now GENERATED from the schema, so it can never drift again.
- The `/admin` panel is REMOVED, and with it the DB override layer that sat above `.env`: it configured the stack from inside the stack, its stale rows silently shadowed `.env` both ways, and its runtime hot-swap could disagree with what compose had actually started. Config is now one layer: `.env` in, `env -> default` resolution, applied on the next compose up. Leftover override rows in old databases have no effect; `gamentic-setup doctor` spots and clears them.

### The Anna agent becomes the anna-mode brain
- The vendor's Anna CLI agent now runs inside the stack (`infra/anna-agent/`), and a thin adapter (`infra/anna-api/`) gives it an OpenAI-compatible face: the orchestrator's tool-calling rides a JSON contract over the agent's streaming copilot API, stops are cut client-side, and the adapter authenticates by riding the agent's own Web UI sign-in (the refresh token on its volume), so no password ever touches `.env`. Sign in once at `http://localhost:19001`; proven live with a played adventure (narrator prose, character dialogue, tool calls, state intact). Images answer 501 and the game keeps playing text-only by design.
- `./up.sh`: the mode-aware launcher. Reads `ANNA` from `.env`, starts the matching stack, cleans up the other mode's leftovers when the boolean was flipped without a down, and refuses non-literal `ANNA` values (which would start no inference services at all). `./up.sh down` stops everything across both modes.

### Phones stop shattering
- One additive `@media (max-width: 640px)` block: the composer wraps into rows, prose-measure badges and bubbles cap at the panel instead of `64ch`, the affordance board stacks, and the story keeps a guaranteed reading window. Desktop verified byte-identical in a real browser; a lint-style test pins the adapter so it cannot silently leak into desktop rules.

### Anna mode: one boolean, zero local inference
- For the Anna hackathon (Jun 15-22): `ANNA=true` in `.env` runs the whole game with nothing GPU or inference-shaped on the box. `docker compose up` starts only the orchestrator and the frontend (the text model, ComfyUI, the image adapter and both voice services are profiled out via `local-inference-anna-${ANNA}` plus a pinned `COMPOSE_PROFILES`); text and image are served by Anna's OpenAI-compatible cloud gateway (`ANNA_API_KEY`, `ANNA_BASE_URL`, `ANNA_TEXT_MODEL`, `ANNA_IMAGE_MODEL`, one base URL normalized per modality); voice is off, the game plays text-only.
- The `/admin` panel grew an Anna card (yes/no, write-only API key, base URL, models, Test buttons). Superseded the same day: the panel and its override layer were removed in favor of the setup faces above; the boolean now lives in `.env` alone.
- Expansion, not restriction: anna off resolves byte-identical to the local stack (pinned by test), per-modality settings wait underneath untouched, and character voice designs are still composed at creation so a cloud-born adventure keeps its voices for a later local life.
- Hardened by an adversarial review round: the app parses the ANNA boolean exactly like compose's literal profile match (`ANNA=1` can never strand the app in local mode while compose skipped the GPU containers), a missing `COMPOSE_PROFILES` line fails loudly with the fix in the message instead of silently dropping the local stack, and the mode-switch rule (compose down before flipping) is written where it is needed.

### Who is who (owner live finds in "Shadows of the Eternal Night")
- A whispered request for a weapon got a flawless in-prose handover ("pulls out a heavy, blackened iron revolver... slides it across the table") and the pack never changed: the character narrated give_item instead of calling it. The ability was always there (a character can produce a plausible item on the fly and hand it to the player, whisper channel included, receipt private); now the prompt demands the call with the same force as attack (prose never moves an object), shows a worked give example, and give marks written as TEXT ("[give_item, a worn brass key", "{give_item: a coin to Bron}") lift exactly like memory marks - a real call and a mark in prose land identically. Bare prose in brackets stays narration; nothing is ever invented from description alone (owner rule: the game does it, not a hard fix).

Every character call now states the roster outright - WITH YOU IN THE SCENE: the hero, then every other present character by name, none of them ever you - and the prompt pins the two channel truths: a PLAYER line is the hero speaking in their own voice whoever it names, and in a private exchange EVERY word stays inside it (the room hears nothing and cannot be addressed).

An old grammar rewording had flipped the direction of both the receipt and the recorded moment (which then fed the character's own memory ever after). Direction-true now: "Leyla is an acquaintance to you now." / "Became an acquaintance to the player", and set_relation's contract says plainly it is the bond toward the player, never the character's own title (the same slip had set a politician's relation to "respected politician").
- The deck sleeps while the narrator works (owner call): the scene name, the goal/quest chip and every item slot lock during a turn alongside the action buttons; the lightbox and profiles stay live. A deliberate reversal of the old "read-only stays live everywhere" rule, for the deck taps only.

## 2026-06-11

### Every file dies with its adventure (owner rule: no timers, ownership only)
- The ComfyUI staging folder used to accumulate forever (448 orphaned renders, 138 MB, including adventures deleted long ago). The owner rejected retention timers outright, so deletion is ownership now: the orchestrator deletes each staging file the INSTANT its own /media copy is persisted (the one failure case - a download that fell back to serving the staging URL - leaves its path in the DB, and game-delete sweeps those); image-api grew the path-safe DELETE endpoints (single file + confirm-gated purge-all) over the staging mount.
- Audio became attributable: every speak carries the active game_id, voice-api keeps a filename-to-games manifest beside the wavs (atomic, recorded on cache hits, refcounted - a line two games cached survives until its last owner dies), and DELETE /voice/games/{gid} kills a game's wavs. Deleting an adventure now removes its DB rows, its /media folder, its staging leftovers and its audio; "Wipe all memory" additionally empties the ENTIRE staging dir and wav cache, so nothing generated survives anywhere, owned or orphaned. Four suites cover it (orchestrator, frontend, image-api, voice-api), built by four parallel lanes.
- Found while verifying: the backend test suite itself had been quietly rendering REAL images (a default localhost:9001 reached the live service from tests that enable images without stubbing media). The suite now points media at a dead port - hermetic, and 95 seconds collapsed to 7.

### Gifts answer in whispers (owner playtest, round three: the give flow becomes intimate)
- The public echo leaked a raw inventory id ("you give 408f0801a83d to Sera"): the give picker rightly sends the item ID, but the echo only knew entity-chip names. The compose seam now resolves bare ids against the DB (pack item id to its stored name, character id to their name), so echoes show names whatever a client sends.

The frontend rides along: when a give resolves, you are redirected straight into that character's Whisper tab, scrolled to their reply.

The overloaded inner [whisper] (the voice-tone idiom inside a say) still reads as tone, never as a private span.

No new timers: badges update on the same turn-response and SSE-refetch paths everything else uses.

### The exigent audit pays off (a wipe, 40 played turns, every narrator/character pair read)
- A full end-to-end audit on a fresh adventure (plus a 24-agent static review of the code) confirmed the scaffold fixes hold under fire - zero think-spans, tool syntax, scaffold labels or narrator-voiced dialogue across 191 stored beats - and surfaced a new top defect: the narrator narrates travel without ever calling move_location, and one missed move corrupts everything downstream (scene descriptions overwritten, items pinned to the wrong room, characters "witnessing" scenes the player had left).
- Movement is now law, not hope: a do-text naming a revealed exit moves the player in the ENGINE before the narrator speaks, the narrator prompt gets a movement-first rule for free-text journeys, and a turn that walks a stayed character along in prose feeds the discrepancy back to the narrator next turn (set_following them or write them out). A death turn now always ends lost no matter what the model says, and the lethal blow is the storyteller's to stage: damage to the player from the narrator is uncapped, while no one can one-shot a character (DAMAGE_CAP, client amounts schema-bounded and clamped at the seam).
- The item economy turns honest: item matching went article-blind ("a rusted lantern" and "rusted lantern" are one thing - no more twin unlock cards, uncleared scene slots or duplicate exits), add_item with a negative qty bounces instead of paying the player, add_item of a visible scene item MOVES it like take_item, remove_item near-misses resolve when exactly one pack item shares the final token, and reveal_item now authors discoveries (scene, character, or straight into the pack) instead of dying on items never planted hidden. A spawned character absorbs their scene-item ghost (the sleeping camel driver no longer exists twice).
- Receipt hygiene: no-op disposition calls are silent ("Tamsin turns hostile." printed six times while she already was), the empty-to-stranger relation is bookkeeping (no more "Tamsin is now your stranger." or fake pivotal moments), relation receipts read with grammar ("Basir now sees you as a reluctant ally."), rejection copy is article-safe, and a whisper to a dead, absent or unknown character bounces with a friendly beat instead of returning silence.
- Every model-text path is scrubbed now, not just narration: character replies (think spans deleted before tag parsing, never reclassified into visible action beats), recap folds, /explain and the creator chat; a markup guard kills HTML dumps (a raw div panel of exits reached a stored beat, live); first-line screenplay impersonation dies; the over-eager pseudo-tool rule no longer eats lines like "We attack (quietly) at dawn"; wishes are fenced so player text cannot fabricate prompt blocks; render() no longer re-expands placeholders inside substituted values.
- Creation makes the fiction true in state: possessions the chat established are seeded into the pack (the sealed ledger that existed only in prose is the canonical incident), the story clock starts when the fiction says (a rainy-evening opening no longer starts at 10:00), every character card ships a one-line description, player_life is code-clamped (the creator once dealt a 1-HP hero), and the finalize prompt now keeps the player's secrets OUT of the public opening scenario (a "secret" map case written into the opening prose had landed in every bystander's memory).
- Replayed end to end on the fixed build: movement fired on the first try in both forms, the seeded item and night clock were live at turn 0, leak probes stayed clean through the recap fold, amount=9999 became a 422, a 999 became a capped blow, the room now reacts to violence (scene status, interventions), and the story can actually be lost. Suite grew 342 to 450 deterministic tests.

### The scaffold regression, killed at the root (live find by the owner)
- At deep context the model regressed into printing its turn as the worked example's shape - a "(think: ...)" block, a "tools: { ... }" object written as text, a "Prose:" label, even screenplay lines from characters that were never spawned - and called no real tools at all, so the prose moved while the world froze.
- Three causes, all fixed: "reason silently" was physically impossible with hybrid thinking disabled, so narrator reasoning now runs in the model's real thinking channel (on by default, separated from prose by llama.cpp); the worked example itself was a printable template, so it was rewritten to contain no printable format; and scaffold stop sequences now halt a regressing generation at its first marker, letting the resolve pass voice the turn cleanly.
- The display nets were generalized too (reasoning spans strip wherever they start, balanced across lines and nested parentheses; written-out tool blocks die whole), with regression tests pinning the exact raw bytes from the live database. Existing stories were retro-scrubbed clean.

### Inference goes pluggable (owner direction: the engine is ours, the inference is nobody's)
- Each modality now sits behind a provider interface resolved at call time: text (any OpenAI-compatible endpoint), audio (local Maya1, OpenAI TTS, ElevenLabs, fal), image (local ComfyUI templates, OpenAI gpt-image, Google nano banana, fal). Dialects are tiny JSON translators, no SDKs; capability flags (references, seed, emotion mode) make the engine degrade deterministically instead of breaking.
- Character voice identity moved into the engine: the designed voice is composed from the sheet and stored with the character, then resolved per provider deterministically. Switching audio providers re-resolves every voice exactly once; a character never speaks in two voices. The voice service's registry is no longer used and holds no game state.
- A one-file admin panel at /admin (optional ADMIN_TOKEN gate): pick providers, paste keys (masked, write-only, server-side), press TEST per modality, save; the next call uses it, no restart. POST /audio/speak is the key-safe passthrough for cloud audio.
- Honest testing line: local paths are the live-tested defaults; cloud dialects are pinned by contract tests against published schemas and await live verification by anyone holding a key. The README explains the whole abstraction.

### The model swap: Gemma 4 26B-A4B mixture-of-experts
- The text model is now the heretic finetune of Gemma 4 26B-A4B (Q4_K_M, ~16.8 GB): 26B of knowledge, ~4B active per token. Measured on the box: ~900-1000 tok/s prefill and ~55 tok/s decode, against the 12B's ~600 and ~24. A full scripted adventure ran 49 turns with zero timeouts where the 12B lost three; deep-context turns dropped from 60-300s to 20-60s.
- Side-by-side scripted runs (same arc, same code) showed the 26B holds what the 12B could not: no cast impersonation inside narration, state staying in sync with the prose across scene moves, dialogue from the whole cast with richer emotion coverage.

### What the live runs taught the brain (fixes from real play, both models)
- The hybrid model likes to print its reasoning line: a leading "(think: ...)" span is now stripped from narration, never shown. A NARRATOR_THINKING knob can instead route that reasoning into the model's real thinking channel (later made the default; see the scaffold entry above).
- The narrator generation now carries stop sequences for every present character's name, so screenplay-style impersonation ("Vane: ...") halts mid-generation instead of reaching the screen. A line that is nothing but a code-shaped call is scrubbed whatever its name (a hallucinated tool once leaked as prose).
- Failed tool calls get a second chance and a voice: a call that failed only because of ordering inside the same reply (a disposition set before its own spawn) is retried once and lands; whatever stays invalid is reported to the narrator next turn ("fix the call, not the story") instead of vanishing silently.
- A turn is a beat, not a chapter: at most two characters speak per turn and each acts once, so quest unlocks, scene changes and new voices land as moments instead of pile-ups. Both dials are player-facing per-game settings now (voices per turn, acts per voice), next to the story-memory ones.
- Thin character backstories get a real biography at creation: one focused call per character writes 5-8 concrete sentences (the whole-world finalize call consistently under-delivered), guided by a worked example. Trait wording is steered to vivid behavior words (cynical, impulsive, fiercely loyal), and the trait-fragment bug turned out to be the prompt's own worked example teaching a dangling word - fixed at the source.
- Snapshot images get unique filenames (two background renders could overwrite each other and leave two captions pointing at one image); a closing emotion tag can no longer leak into dialogue as text; a length-cut fragment with no finished sentence is dropped instead of displayed; the transport timeout honors the slow-turns decision (300s).

### Frontend (round 3.5: the stream listens, the keyboard comes back)
- Blind media polling is gone: one EventSource per open game listens on GET /games/{gid}/events and re-fetches /state (scene art, portraits) or /beats?since= (item cards, late image beats) the moment the backend announces a persisted render. A reconnect owes one catch-up fetch of both; a slow 60s sweep stays for proxies that break SSE. The look placeholder now simply waits for its event, and the 40s-poll-ceiling bug class (a scene render landing at +47s, recoverable only by F5) is structurally dead.
- Cloud audio readiness, contract pinned: voice playback now tolerates BOTH speak shapes, { audio_url } from the local voice-api (unchanged default) and raw audio bytes from the orchestrator's key-safe passthrough (blob to object URL). The nginx /voice proxy target is an env template (VOICE_UPSTREAM / VOICE_PATH_PREFIX), so flipping to a cloud provider is a container env change, not a rebuild.
- The whole image figure opens the lightbox now, caption plate included (the caption was pointer-events-none chrome and clicks on it died). Character names everywhere they label something (speech bubbles, scene columns, cast rows) wear the display type like the image titles.
- An arriving image ALWAYS scrolls into view (the old near-bottom rule with its "new image below" chip left late renders unseen), late SSE arrivals included, and image loads keep the reader pinned to the newest content instead of shifting them mid-history.
- When a turn or creator reply finishes, focus returns to the active input on its own (main composer, whisper composer, or the creator chat box), never stealing from a control the player is actively using.
- 186 tests across 10 files, was 179: the SSE scene/beat/reconnect flows, the no-SSE fallback sweep on a fake clock, both speak shapes, the whole-figure lightbox, the three focus returns.

### Frontend (round 3.4: the render becomes a morph)
- render() no longer rebuilds the DOM with innerHTML: it morphs the real DOM against the fresh HTML (vendored idiomorph, one 0BSD-licensed file, still no build step). Unchanged nodes keep their identity, so focus, caret, scroll positions and mid-flight animations survive background re-renders structurally; the hand-rolled scroll bookkeeping is gone and the typed-input snapshot demoted to a belt-and-braces for plain inputs.
- Events went with it: per-element re-binding after every render is replaced by five delegated listeners on the root, attached once. An in-place pane patch needs no re-wire, and preserved nodes can never double-fire.
- Verified live against the running stack, not just in jsdom: typed text and composer focus survive profile open/close renders in a real browser, a full narrator turn (echo, reveal, prose, art) plays correctly under morphing, zero console errors. 179 tests.

### Engine and state machine (the brain)
- Agentic input interpreter: freeform typed actions are parsed by the model into structured say/do/attack/give/whisper segments (grounded in who is present and what you carry), so plain typing gets the same directed routing, private whispers and adjudication as the composer buttons. Bounded and validated; any failure falls back to the raw text. `INTERPRET_FREE_TEXT` env, on by default.
- Ask what this is: `POST /games/{id}/explain` gives an in-world, 2-3 sentence explanation of any tapped thing (item, character, scene, quest, goal, or a system receipt beat), built from player-visible facts only, so it can never spoil hidden items or character secrets.
- Receipt polish: duplicate tool calls in one narrator reply are suppressed (no more doubled "Obtained" items), model-invented snake_case names are humanized everywhere the player sees them, and quest/objective receipts say WHAT changed ("Objective complete: Reach the wall.").
- SQLite hardened for concurrency: WAL mode plus a generous busy timeout, so background image persists queue behind a running turn instead of erroring "database is locked".
- Narrator reasons about every state transition internally (state now, what happened, next state) and changes the world only through validated tools; the database is the single source of truth.
- Resolver-style prompt dispatch: a lean narrator core, plus situational protocol blocks (furnish a new place, return after an absence) injected only on the turns that trigger them, each carrying a few-shot example. Few-shots also added for adjudication, character tool use, and image description fields.
- Player attempt adjudication: impossible attempts are rejected deterministically with friendly in-world reasons; valid attack/give attempts go to the narrator to accept, adjust, or veto with a reason; anything left untouched happens as attempted, so no action is ever silently dropped. The veto tool is only offered while attempts are pending.
- Fictional story clock (hybrid): a few minutes pass automatically per action, the narrator jumps it for rests and journeys, and day/part-of-day derive from it.
- Draft persistency layer: leaving a scene stamps when, the narrator can note open threads, and returning surfaces the elapsed time plus the note so the place plausibly lived while you were gone.
- Entity chips: characters and items tagged in the composer resolve by real id, so names never drift.
- Private 1:1 channel: whisper say/do exchanges that other characters never see, composed as one exchange with one reply.
- Canonical scene keys (fixes a live bug where location-name drift duplicated scenes) and output hygiene scrubbing for small-model artifacts.
- Context meters: a global one (the biggest agent prompt of the turn) and one per character agent, since each character runs in its own context.

### Images
